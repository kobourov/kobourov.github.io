In this section we'll discuss how you can manipulate images in .

Viewport
Spacing
Scaling - \scalebox{horizontal factor}[vertical factor]{item scaled}
Resizing - \resizebox{horizontal factor}{vertical factor}{item resized}
Rotating - \rotatebox{angle of rotation}{item rotated}
Reflecting - \reflectbox{item reflected}
Centering - \begin{center} centered items \end{center}

Note that these commands all work with text as well - diagrams are not the only items that can be reflected, rotated, etc. In practice, the centering is the only one of the above you're likely to use much with text.

We'll start where we left off on the How To page. If you haven't already reviewed that page, do so now. Make sure you save the myimage.pdf file to your computer for use in these examples. Our original source code is:

\documentclass[11pt]{article}
\usepackage[pdftex]{graphicx}
\begin{document}

This is my first image.

\includegraphics*[viewport=100 200 500 650]{myimage}

That's a cool picture up above.

\end{document}

Viewport

The viewport portion of the \includegraphics* command tells what portion of the myimage file to include. The first two coordinates are the (x,y) coordinates in pixels of the lower left corner of the portion of the myimage file you want to inclue. The second two coordinates are the upper right values of (x,y). Try changing the values in the original source file and see what happens.

\documentclass[11pt]{article}
\usepackage[pdftex]{graphicx}
\begin{document}

This is my first image.

\includegraphics*[viewport=200 400 400 650]{myimage}

That's a cool picture up above.

\end{document}

For example, the code above should result in an image like this. The lower left corner needs to be moved lower and more to the left. The y-coordinate of the upper right corner is probably ok, but the x-coordinate needs to be increased.

Unfortunately, there's no easy way to read off the coordinates for your image. You'll have to use trial and error to set these values for each image.

Spacing

One issue to watch out for is spacing. Suppose we take out the empty lines from our original source code:

\documentclass[11pt]{article}
\usepackage[pdftex]{graphicx}
\begin{document}

This is my first image.
\includegraphics*[viewport=100 200 500 650]{myimage}
That's a cool picture up above.

\end{document}

If you compile this, you should get something that looks like this. By removing the blank lines, we are telling to try to put the image in the same line as the text. You see the mess that results - there's not enough room in the line for the image, and if we had paragraphs before and after the image, the formatting could become very messed up. Generally, put an extra line before and after your image inclusion in your source file and you should be fine. This will also help you find the image in the source file if you need to edit.

You can also use \vskip and \hskip or other spacing commands to move images around. Compare the results of the source code below to the original source code results to see the results of using \vskip and \hskip.

\documentclass[11pt]{article}
\usepackage[pdftex]{graphicx}
\begin{document}

This is my first image.

\vskip-2in
\hskip4in
\includegraphics*[viewport=100 200 500 650]{myimage}

That's a cool picture up above.

\end{document}

The \hskip with the positive 4in (in means inches) moved the diagram far to the right. The \vskip with the negative 2in moved the diagram up (and everything after it as moved up as well).

Scaling

We can easily scale the size of the diagram by using the \scalebox command.

\documentclass[11pt]{article}
\usepackage[pdftex]{graphicx}
\begin{document}

This is my first image.

\scalebox{0.40}[0.40]{\includegraphics*[viewport=100 200 500 650]{myimage}}

That's a cool picture up above.

\end{document}

In general, we use \scalebox by putting the horizontal scaling in the initial { }, the vertical scaling factor in the next [ ], then the item to be scaled in the final { }:

\scalebox{horizontal factor}[vertical factor]{item scaled}

Experiment with your source file to see how \scalebox works - try changing the 0.40s to different numbers and check out the effects when you compile. The source above should give you something like this.

Resizing

We can resize a diagram by using the \resize command.

\documentclass[11pt]{article}
\usepackage[pdftex]{graphicx}
\begin{document}

This is my first image.

\resizebox{5in}{2in}{\includegraphics*[viewport=100 200 500 650]{myimage}}

That's a cool picture up above.

\end{document}

In general, we use \resize by putting the horizontal size in the initial { }, the vertical size in the next { }, then the item to be resized in the final { }:

\resizebox{horizontal factor}{vertical factor}{item resized}

Experiment with your source file to see how \resize works - try changing the dimensions to different numbers and check out the effects when you compile. The 'in' after the 5 and 2 say the dimensions are in inches. You can also use cm or pt, among others; if you use no dimensions, the result will be in pixels. The source above should give you output that looks like this.

Rotating

We can rotate a diagram by using the \rotatebox command.

\documentclass[11pt]{article}
\usepackage[pdftex]{graphicx}
\begin{document}

This is my first image.

\rotatebox{45}{\scalebox{0.70}[0.70]{\includegraphics*[viewport=100 200 500 650]{myimage}}}

That's a cool picture up above.

\end{document}

In general, we use \rotatebox by putting the angle of rotation in the initial { }, then the item to be rotated in the next { }:

\rotatebox{angle of rotation}{item rotated}

Experiment with your source file to see how \rotatebox works. Notice in the above example that we used it in conjunction with \scalebox, rotating a scaled image. In general, you can combine all these scaling, resizing, etc., commands by nesting them as we've nested the \scalebox inside \rotatebox above. The above source should yield a rotated image like this one.

Reflecting

We can horizontally reflect a diagram by using the \reflectbox command.

\documentclass[11pt]{article}
\usepackage[pdftex]{graphicx}
\begin{document}

This is my first image.

\reflectbox{\includegraphics*[viewport=100 200 500 650]{myimage}}

That's a cool picture up above.

\end{document}

\reflectbox is pretty simple - put the item to be horizontally reflected in { }.

\reflectbox{item reflected}

The code above should give you something like this.

Centering

We center text or images using \begin{center} and \end{center}.

\documentclass[11pt]{article}
\usepackage[pdftex]{graphicx}
\begin{document}

This is my first image.

\begin{center}
\scalebox{0.40}[0.40]{\includegraphics*[viewport=100 200 500 650]{myimage}}
\end{center}

That's a cool picture up above.

\end{document}

Just put \begin{center} when you want to start centering, and \end{center} when you want to stop centering. (If you want to center everything until the end of the document, you still need an \end{center} before the \end{document} at the end of the source file.

The code above should give you something like this.

If you use this centering, but your diagram is still off-center, then you probably have too much white space included with your diagram. Try adjusting the x-values in the viewport line. To see an example of this problem, change the 500 above to 750. You should end up with something like this. The image is centered, but there is a whole lot of blank space included in the image, thus making the image look off-center.
